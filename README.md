## O que este projeto demonstra

| Competência | Aplicação prática |
| --- | --- |
| API testing | Requisições organizadas para validar comportamento de serviço |
| Postman | Coleção compartilhável e executável via botão oficial |
| Documentação | Contexto claro para quem abre o repositório |
| QA em camadas | Validação de serviço sem depender da interface |

## Como usar

1. Clique em `Run in Postman`.
2. Importe ou faça fork da coleção.
3. Configure variáveis de ambiente, se a API exigir.
4. Execute as requisições e valide status, payloads e mensagens retornadas.

## Resultado técnico

Este repositório evidencia uma prática essencial em QA: testar APIs de forma direta, organizada e reprodutível. Coleções Postman bem estruturadas aceleram investigação, documentação de contrato e comunicação com times de backend.
